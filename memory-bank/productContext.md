# Product Context: JBFH

## Зачем проект существует
Детские хоккейные школы в Беларуси нуждаются в централизованной системе для управления:
- Клубами и их филиалами
- Командами разных возрастов
- Тренерским составом
- Конструктором упражнений (лёд/земля)
- Инвентарём

## Проблемы, которые решает
- Разрозненность данных о клубах и командах
- Отсутствие единой базы тренеров и их назначений
- Нет системы учёта инвентаря и упражнений
- Ручное управление доступом (решается ролевой моделью)

## Как должна работать система
1. **Администратор** создаёт клубы, назначает пользователей
2. **Клуб** управляет своими командами, тренерами
3. **Методисты** (общие и клубные) просматривают данные
4. **Тренеры** видят свои команды и упражнения
5. **Конструктор упражнений** — все могут создавать инвентарь и упражнения, но редактируют только ADMIN/METHODIST/CLUB/CLUB_METHODIST

## User Experience цели
- Простой REST API с пагинацией на всех списках
- Swagger UI для ознакомления и тестирования
- Загрузка файлов через multipart (логотипы, картинки)
- Мягкое удаление (active=false) вместо физического
- Dynamic enum endpoints (GET /types) для фронтенда
