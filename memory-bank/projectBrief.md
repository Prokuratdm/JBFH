# Project Brief: JBFH

## Название
JBFH — Hockey Schools Management API

## Цель
REST API для управления детскими хоккейными школами в Беларуси. Система позволяет управлять клубами, командами, пользователями с ролевой моделью, содержит конструктор упражнений с инвентарём, конструктор тренировок с шаблонами и сетами, программы обучения, учёт детей и их физических показателей.

## Ключевые требования
1. **JWT-аутентификация** — Bearer token, HS256
2. **Ролевая модель** — 6 фиксированных ролей (ADMIN, CLUB, METHODIST, CLUB_METHODIST, COACH, MAIN_COACH)
3. **Управление клубами** — CRUD, логотипы, пагинация
4. **Управление командами** — вложенные в клубы, назначение тренеров, деактивация вместо удаления
5. **Конструктор упражнений** — инвентарь (общий/клубный), упражнения с типами (ICE/LAND), картинки, связь many-to-many с инвентарём, классификация (TrainingPart, Focus, PreparationType)
6. **Конструктор тренировок** — тренировки с упражнениями, авто-расчёт restDuration/workMode/totalTime через ExerciseCalculator, шаблоны тренировок, сеты упражнений для фаз
7. **Загрузка файлов** — логотипы клубов/команд, картинки упражнений (через FileStorage абстракцию)
8. **Swagger-документация** — OpenAPI 3.0
9. **Тесты** — юнит-тесты (Mockito) + интеграционные (Testcontainers)

## Scope
- **Включено**: Auth, Users, Clubs, Teams, Inventory, Exercises, Trainings, TemplateTrainings, Sets, TrainingPrograms, Children, ChildPhysicalStats, ChildStandards, Standards, FileStorage
- **Исключено**: Расписания, турниры, статистика (на будущее)
- **Нельзя удалять**: клубы, команды, инвентарь, упражнения — только деактивация
- **Можно удалять**: тренировки, сеты, дети
